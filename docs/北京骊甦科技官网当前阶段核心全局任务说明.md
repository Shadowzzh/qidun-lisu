## 当前阶段的核心全局任务

当前阶段的核心全局任务，是基于 `archive/ppt-source` 中现有的文案、结构化内容和图片素材，将 `ai-website-cloner-template` 逐步演进为“北京骊甦科技”的正式多页面官网。

这项工作的目标，不是只完成一个首页，而是建立一套可持续扩展的官网基线。该基线需要同时覆盖品牌主体、方案表达、能力页面、案例页面、团队页面，以及与之配套的导航体系、内容组织方式和运行时资源体系，使后续页面能够在同一套结构下继续扩展，而不是每增加一个页面就重新定义一套布局和交互。

在页面结构和视觉组织上，全站需要保持统一的站点壳和页面秩序。各页面应共用同一套 `Header` 与 `Footer`，并保持一致的导航层级、版心宽度、间距节奏、响应式规则和交互行为。不同页面可以承载不同主题内容，但不应在布局逻辑、外层骨架和整体阅读路径上彼此割裂，避免出现“每个页面都像不同网站”的问题。

在布局基线方面，应尽量沿用模板已有的页面骨架和 section 节奏，例如首屏主视觉区、总览或摘要区、图文能力区、案例或收束区、页脚区等。这种统一的框架不仅是为了视觉一致性，也是为了保证后续扩展到更多页面时，整站仍然保持清晰、稳定、可维护的结构。

在内容来源上，网站必须以 `archive/ppt-source` 作为唯一归档基础来源。不新增外部业务资料，不伪造不存在的新闻、客户、备案、联系方式等信息。图片素材优先使用 `archive` 中已有内容；如果某一页或某一区块没有合适图片，应使用说明型占位，而不是随意补充不具备来源依据的视觉内容。

最终交付目标，是让整站呈现为“北京骊甦科技的正式官网”，而不是“替换了文案的模板站”，也不是“把 PPT 内容拆成多个网页的展示稿”。
